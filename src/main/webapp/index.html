• Big-O Chart

The goal is to visualize Big-O trends on the chart below. Specifically, we are testing Java collections. You can compare different Java collections and operations. The backend benchmarks the time for an operation to complete on a collection up to a max input size. Input size is the number of elements in the collection when the operation runs. It returns raw data, so you will see best, worst, and average case times plotted in the chart. Even with raw data, you can see the upper range, representing worst case scenarios, resulting in the Big-O trends we might expect to see.

There are going to be outliers in the data. Also, some operations are so quick that they only take tens of nanoseconds to perform. There is no filtering on the data, it's raw data, so it may be a good idea to run these tests multiple times. Again, the goal is to visualize the trends, so even with results including outliers and very fast operations we can still identify if an operation is linear or constant time.

The official documentation for Java collections can be found at Java Platform, Standard Edition & Java Development Kit, Version 11 API Specification. Pieces of the documentation are included here for convenience. You can view these summaries by clicking on the Info button of a configured series.
